ARCHITECTURE OVERVIEW
───────────────────────────────────────────────────────────────────────────────

Route Structure:
- /admin                    → Dashboard (if authenticated)
- /admin/login              → Login page (email/password only)
- /admin/inquiries          → Inquiry manager (table, filters, status updates)
- /admin/packages           → Package CRUD
- /admin/gallery            → Photo/video manager
- /admin/reviews            → Testimonials manager
- /admin/analytics          → GA4 embed, WhatsApp clicks, form submissions
- /admin/messages           → WhatsApp template composer

Authentication (Supabase):
- Supabase Project URL: Create at https://supabase.com/dashboard
- Service role key: Keep secret (server-side only)
- Admin table: admin_users (email, is_active, created_at)
- Session: httpOnly JWT cookie (NOT localStorage—prevents XSS)
- Timeout: 4 hours, then re-authentication
- Failed logins: Lock after 5 attempts for 30 minutes


1. CREATE AUTHENTICATION SYSTEM
──────────────────────────────────────────────────────────────────────────────

A. Create Supabase tables:

   Table: admin_users
   ┌─────────────┬──────────┬─────────────────────┐
   │ Column      │ Type     │ Notes               │
   ├─────────────┼──────────┼─────────────────────┤
   │ id          │ uuid     │ Primary key         │
   │ email       │ text     │ Unique, indexed     │
   │ password    │ text     │ Bcrypted (Supabase) │
   │ is_active   │ boolean  │ Can they log in?    │
   │ created_at  │ timestamp│ Account creation    │
   │ last_login  │ timestamp│ Last access         │
   │ failed_attempts │ int  │ For lockout logic   │
   │ locked_until│ timestamp│ Lockout expires     │
   └─────────────┴──────────┴─────────────────────┘

   Seed with Steve + any co-managers:
   - steve@natureloversadventure.co.ke
   - [Additional managers as needed]

B. Create src/routes/admin/login.tsx

   Features:
   - Email + password form (no social login, no "forgot password" link publicly visible)
   - Error handling (invalid credentials, account locked, server error)
   - Redirect to /admin if already authenticated
   - Button: "Request Password Reset" (admin sends via email/WhatsApp)

C. Create authentication middleware:

   src/lib/auth.ts:
   - verifyToken(jwt): Check if valid, not expired
   - loginAdmin(email, password): Hash comparison, update last_login
   - logoutAdmin(): Clear httpOnly cookie
   - lockAccount(email, duration): Set locked_until timestamp
   - checkFailedAttempts(email): Increment counter, lock if > 5

D. Set httpOnly cookie on login:
   
   res.setHeader('Set-Cookie', `auth_token=${jwt}; HttpOnly; Secure; Path=/; Max-Age=14400`);
   // 4 hours = 14400 seconds

   import { useEffect, useRef } from 'react';
   
   export function useRipple() {
     const ref = useRef<HTMLElement>(null);
   
     useEffect(() => {
       const element = ref.current;
       if (!element) return;
   
       element.addEventListener('mousedown', (e: MouseEvent) => {
         const rect = element.getBoundingClientRect();
         const x = e.clientX - rect.left;
         const y = e.clientY - rect.top;
   
         const ripple = document.createElement('span');
         ripple.className = 'ripple';
         ripple.style.left = `${x}px`;
         ripple.style.top = `${y}px`;
         element.appendChild(ripple);
   
         setTimeout(() => ripple.remove(), 600);
       });
     }, []);
   
     return ref;
   }
